## Overview

HybStockAdvisor is a mobile-first Flutter application designed to help Nigerian investors make informed decisions about the stock market. The app connects to a Python FastAPI backend (port `8000`) that runs AI-powered forecasting models and market data aggregation. Users can build and track a personal portfolio of NSE-listed stocks, receive AI-generated buy/sell/hold recommendations, chat with an AI assistant named **Lexi**, and view detailed market analytics — all within a clean, themeable UI.

## Features

### Authentication

- **Login** — Email & password authentication with JWT token storage.
- **Signup** — New user registration with full onboarding flow.
- **Forgot Password** — Initiate password reset via email.
- **OTP Verification** — One-time password verification screen for identity confirmation.
- **Reset Password** — Set a new password after OTP verification.

### Dashboard

- **Live Market Data** — Real-time prices and percentage changes for top NSE-listed stocks (e.g., DANGCEM, GTCO, MTN Nigeria, Zenith Bank, SEPLAT, BUA Foods, Airtel Africa, UBA, etc.).
- **Market Summary** — Overview of market-wide performance.
- **Stock Charts** — Sparkline and detailed price charts powered by `fl_chart` and `syncfusion_flutter_charts`.
- **Search** — Search for any listed Nigerian stock.

### Portfolio

- **Holdings Tracker** — Track stocks you own with current price, quantity, average buy price, and percentage gain/loss.
- **Watchlist** — Monitor stocks of interest without owning them.
- **Sparkline Charts** — Per-stock mini trend lines showing recent performance direction.
- **Add / Remove Holdings** — Manage portfolio entries directly from the app.

### AI Insights

- **AI Recommendation** — Buy, Sell, or Hold signals generated by the Python AI backend for any selected stock.
- **AI Confidence Score** — A percentage confidence indicator for each recommendation.
- **Market Stability Score** — Measures how stable the broader market environment is.
- **Public Sentiment Score** — Aggregated sentiment indicator for the chosen stock.
- **Key Market Drivers** — Narrative explanation of the factors influencing the recommendation.
- **Price Forecast Chart** — Forward-looking price prediction visualisation.

### AI Chat Assistant — Lexi

- A conversational AI chat interface powered by **Google Gemini** via the Python backend.
- Available as a bottom sheet from any screen.
- Context-aware: when opened from a stock detail view, Lexi is pre-seeded with the ticker context for relevant answers.
- Supports free-form questions about Nigerian stocks, market trends, and portfolio advice.

### Notifications

- **In-App Notification Centre** — Persistent list of alerts stored locally via Hive.
- **Unread Badge** — Notification count indicator on the bottom navigation bar.
- **Mark as Read / Mark All Read** — Manage read state of individual or all notifications.
- **Swipe to Dismiss** — Remove individual notifications with a right-to-left swipe.
- **Deduplication** — One notification per type per calendar day to avoid spam.

### Profile

- **Display Name** — Shows the user's first and last name retrieved from Hive local storage.
- **Avatar** — Supports custom profile photo set via image picker; falls back to a default asset.
- **Quick Links** — Navigate to Personal Info, Settings, and Notification Settings from the profile page.
- **Logout** — Clears auth token and navigates back to the login screen.

### Personal Info

- Edit user profile details (first name, last name, etc.) synced with the backend.
- Upload and crop profile avatar image via `image_picker`.

### Settings

- **Dark / Light Mode Toggle** — Switch between dark and light themes; preference persisted to Hive.
- **Clear Cache** — Clear locally cached data with a confirmation dialog.
- **App Version Info** — Displays current version via `package_info_plus`.

### Notification Settings

- Configure which categories of notifications are enabled.

### First Login / Portfolio Setup

- First-time users are guided through a portfolio setup wizard after initial login.
- The wizard fetches the live NSE stock list from the backend.
- Users select stocks they own, enter quantity and average buy price, and submit to the backend to initialise their portfolio.
- If the backend reports an existing portfolio on login, this step is skipped automatically.

## Architecture

```
Flutter App (Dart)
        │
        ├─ UI Layer (Screens & Widgets)
        │        └─ Provider (State Management)
        │
        ├─ Service Layer
        │        ├─ ApiService      →  Python FastAPI backend (port 8000)
        │        └─ RequestService  →  Secondary REST API (port 3001, legacy/auth)
        │
        └─ Local Storage
                 └─ Hive (auth tokens, user data, notifications, theme prefs)
```

The app follows a **Provider + Service** pattern:

- **Providers** hold reactive state (theme mode, notifications).
- **Services** handle all network I/O with `Dio` and token injection via interceptors.
- **Screens** read from providers and call service methods, keeping business logic out of widgets.

| --------- | ----------- | ------------------ |
| 0         | Dashboard   | Home / Market      |
| 1         | AI Insights | Insights / Sparkle |
| 2         | Portfolio   | Portfolio / Chart  |

**Navigation flows:**

- Any screen → `NotificationCenter` (via bell icon)
- Any screen → `AiChatSheet` (via Lexi FAB or chat icon)

## State Management

The app uses the **Provider** package with two `ChangeNotifier` classes:

### `ThemeProvider`

- Stores the active `ThemeMode` (`light`, `dark`, or `system`).
- Exposes `toggleTheme(bool isDark)` to switch modes.
- Theme preference is persisted to Hive (`user` box, key `theme_dark`).

### `NotificationProvider`

- Maintains a sorted list of `AppNotification` objects.
- Persists notifications to Hive (`notifications` box).
- Provides:
  - `addNotification()` — with same-day deduplication per `NotificationType`
  - `markRead()` / `markAllRead()`
  - `removeNotification()`
  - `clearAll()`
  - `unreadCount` getter — drives the badge on the nav bar

## Services & API Layer

### `ApiService` (Primary — Port 8000)

Connects to the **Python FastAPI** backend. All endpoints are under `/api`.

| Method                     | Endpoint                  | Purpose                            |
| -------------------------- | ------------------------- | ---------------------------------- |
| `getStockForecast(ticker)` | `GET /forecast/{ticker}`  | AI price forecast for a stock      |
| `getMarketSummary()`       | `GET /summary`            | Full NSE market snapshot           |
| `getInsights(ticker)`      | `GET /insights/{ticker}`  | AI insights + recommendation       |
| `sendChatMessage(text)`    | `POST /chat`              | Lexi AI chat (Gemini-powered)      |
| `login(email, password)`   | `POST /auth/login`        | Authenticate user, return JWT      |
| `register(userData)`       | `POST /auth/register`     | Create new user account            |
| `addToPortfolio(...)`      | `POST /portfolio/add`     | Add a stock holding                |
| `getUserAssets()`          | `GET /portfolio/{userId}` | Fetch user's portfolio & watchlist |

The service reads the `user_id` from Hive (`auth` box) and attaches the JWT via the `Authorization: Bearer` header. Platform-specific host resolution handles Android emulator mapping (`10.0.2.2:8000`).

### `RequestService` (Secondary — Port 3001)

A legacy/auxiliary service client with full Dio interceptor setup:

- Auto-loads the JWT before every request.
- Logs all requests, responses, and errors for debugging.
- Used for additional backend operations separate from the main AI API.

## Local Storage

The app uses **Hive** for all local persistence (no SQLite dependency).

| Hive Box        | Key(s)                                                                        | Purpose                                   |
| --------------- | ----------------------------------------------------------------------------- | ----------------------------------------- |
| `auth`          | `token`, `user_id`                                                            | JWT token and authenticated user ID       |
| `user`          | `first_name`, `last_name`, `avatar_path`, `has_setup_portfolio`, `theme_dark` | User profile and preferences              |
| `notifications` | `list`                                                                        | Serialised list of `AppNotification` maps |

**Secure Storage** (`flutter_secure_storage`) is also included as a dependency for sensitive credential storage when needed.

## Getting Started

### Prerequisites

- Flutter SDK `^3.x` with Dart `^3.9.2`
- A running instance of the Python FastAPI backend on `localhost:8000`

### Installation

```bash
# 1. Clone the repository
git clone <repository-url>
cd hybstockadvisor

# 2. Install Flutter dependencies
flutter pub get

# 3. Run the app (ensure backend is running first)
flutter run
```

### Running on Specific Platforms

```bash
# Android (emulator or device)
flutter run -d android

# iOS (simulator or device)
flutter run -d ios

# Web
flutter run -d chrome

# macOS desktop
flutter run -d macos
```

## Backend Requirements

HybStockAdvisor requires a **Python FastAPI** backend to function. The backend must provide:

- `GET  /api/summary` — Full NSE market price snapshot
- `GET  /api/forecast/{ticker}` — AI price forecast for a given ticker
- `GET  /api/insights/{ticker}` — AI recommendation, confidence scores, and narrative
- `POST /api/chat` — AI chat powered by Google Gemini
- `POST /api/auth/login` — User authentication, returns JWT
- `POST /api/auth/register` — New user registration
- `GET  /api/portfolio/{userId}` — Fetch user portfolio and watchlist
- `POST /api/portfolio/add` — Add a stock to user portfolio

**Host configuration** is automatic:

- Android Emulator → `http://10.0.2.2:8000`
- iOS Simulator / Web / Desktop → `http://127.0.0.1:8000`
